HTML

  • The head element contains information about the webpage. Not visible.
  • The body element represents the visible content shown to the user.
  • Elements: Elements that are nested inside other elements are called child elements. The words “element” and “tag” are commonly used interchangeably in the web development world, but it is important to know the difference. An element refers to the entire element from opening tag to closing tag, whereas a tag refers only to what appears inside the angle brackets.
  • The "ul", "ol", and "li" elements represent unordered lists, ordered lists, and list items.
  • The "img" element contains information about images that are displayed on the webpage. The "img" element can contain various attributes; the src attribute is the most important because it defines the location of the image
  • "br":element creates a line of empty space, or a line break, between two blocks of content.
  • hyperlink: The href attribute points to the URL for the link
  • The `src` attribute defines the location, or source, of the image file.
  • `alt` attribute contains a text string that describes the appearance and functionality of the image. Useful for SEO

Git

  1. git status: checks what branch we are currently on
  2. git checkout main: to go to main branch
  3. git pull origin main: pull down the latest main branch
  4. git checkout -b branch-name: creates a new branch and switches to it
  5. You should see the following message in your terminal: "Switched to a new branch 'feature/add-html'"
  6. Now you are in a the new branch
  7. git status: to confirm which branch your are on
  8. go to VS

Git Flow

  1. git add -A: to commit all files in the working branch.
  2. git commit -m "added code to HTML file": Now that all of our changes have been staged, we can commit them to our repository. Use the command in your terminal to commit the changes. We use the -m flag to associate a message with our commit. Commit messages should be short descriptive messages of the changes implemented. Once completed, our Git repository is now up to date with our working directory!
  3. git pull origin main:It is a good habit to always pull the base branch into your feature branch before pushing up any code. You should see something similar to the following in your terminal if your branches are in sync: Already up to date.
  4. git push origin feature/add-html: push branch
